# DevNote v0.4.0 — Manual E2E Test Checklist

Open side-by-side during testing. Check off as verified. ~40 checks, estimated ~45-60 minutes.

---

## Setup & Upgrade

### Fresh install (no prior DevNote)

- [ ] Install v0.4.0 from `.vsix` on a machine with no prior DevNote
- [ ] Sidebar opens, setup wizard appears (same as v0.3.0)
- [ ] After credentials saved, Idle state shows Generate Doc button + empty Recent Notes + 🔍 search input
- [ ] No Python popups on activation (search is lazy)

### Upgrade from v0.3.0

- [ ] Credentials carry over automatically (no re-entry)
- [ ] All existing v0.3.0 notes still visible in Recent Notes list
- [ ] `devnote.db` file at `globalStorageUri` preserved — schema_version stays at 1
- [ ] Draft banner still works if a v0.3.0 draft existed
- [ ] No Python popup on extension activation — dormant until search
- [ ] Generate + sync a new note → works normally → toast "Note synced. Search indexing on your next search." appears (Python not yet set up)

### Python not installed case

- [ ] On a machine WITHOUT Python, click 🔍 search → popup: "DevNote needs Python 3.10+ [Install Guide]"
- [ ] Click "Install Guide" → opens browser to Python install page
- [ ] Cancel popup → search gracefully disabled, no error state
- [ ] Install Python 3.11, retry search → setup flow proceeds

### Python version too low

- [ ] On a machine with Python 2.7 or 3.7, first search → popup: "Found Python X.Y, need 3.10+"
- [ ] Install Python 3.10+, retry → setup succeeds

### Venv setup flow

- [ ] First search on valid Python → popup: "Setting up Python environment (~30s)..."
- [ ] `vscode.window.withProgress` notification shows "Creating venv..." then "Installing packages..."
- [ ] `globalStorageUri/venv/` directory created with proper structure (bin/python, Scripts/python.exe on Windows)
- [ ] `requirements.txt` packages pip-installed (`google-generativeai`, `numpy` present in `venv/lib/site-packages/`)
- [ ] After setup, `globalState.venvReady = true`

## First-Search Flow (the onboarding moment)

- [ ] First search ever → triggers backfill popup: "Indexing your {N} past notes for search (~{T}s)..."
- [ ] Backfill runs via Gemini batch API (check network tab — should be ~N/100 API calls, not N calls)
- [ ] All existing notes' `embedding` column now populated (verify via SQLite inspect)
- [ ] All existing notes' `embedding_model` = "text-embedding-005"
- [ ] After backfill, search executes immediately (no second wait)
- [ ] Next search has ZERO setup popups — goes straight to results

## Normal Search (warm path)

- [ ] Type a query in 🔍 input → 300ms debounce (no query on every keystroke)
- [ ] Section header morphs from "Recent Notes" to "N results for 'query'"
- [ ] Results render with title + branch + date + percentage score (e.g. `87% · 2h`)
- [ ] Results sorted highest score first
- [ ] Score format is percentage (no decimals visible: `87%` not `0.87`)
- [ ] Top result is semantically relevant (meaning-based, not keyword-based)
- [ ] Search latency <500ms for warm path (worker already running)
- [ ] Irrelevant queries return empty results (below 0.35 threshold)

- [ ] Click ✕ inside search input → clears query → header morphs back to "Recent Notes" → full list restored

- [ ] Click a search result row → opens inline Preview (same as Recent Notes click)
- [ ] Preview shows "Open in Notion" full-width button (v0.3.0 behavior preserved)
- [ ] Click Back from Preview → returns to SEARCH mode with same query preserved (not to Recent Notes)
- [ ] Header still shows "N results for 'query'" after Back

## Sync Flow (embed-on-sync)

### Happy path (Python ready)

- [ ] Generate + sync a new note → Notion sync succeeds → local SQLite row created → embed API call succeeds → `embedding` BLOB populated → toast: "You got it! Note synced to Notion."
- [ ] New note appears in Recent Notes list immediately
- [ ] New note searchable via 🔍 without needing backfill

### Python not ready

- [ ] Generate + sync a note BEFORE first search (Python not yet set up) → toast: "Note synced. Search indexing on your next search."
- [ ] `embedding` column stays NULL for that note
- [ ] First search after this triggers backfill for the NULL note

### Embed API fails

- [ ] Simulate API 429 (rate limit) → note syncs to Notion, embed fails → toast: "Note synced. Search indexing deferred." → NULL embedding persists → next search backfill retries

### Note too long (>2048 tokens)

- [ ] Generate an unusually long note → sync succeeds → embed API returns token-limit error → toast: "Note too long for search indexing. Synced to Notion successfully." → `embedding` stays NULL permanently (until note content ever shrinks)

---

## Error Recovery

### Worker crash

- [ ] Force-kill the Python worker process during a search → toast: "Search worker restarted (1/3)." → auto-respawn → retry search → results appear
- [ ] Force 3 crashes in one session → search disables → error message: "Search worker is unstable. Restart VS Code or check [Python logs]."
- [ ] v0.3.0 generate/sync/Recent Notes flows still work when search is disabled

### Venv setup failure

- [ ] Simulate pip install failure (offline mode or bad network) → toast: "Couldn't set up Python environment: {detail}. [Retry]"
- [ ] Click Retry → setup retries → succeeds when network restored

### Offline scenarios

- [ ] Offline during query → empty state: "Search requires internet (for query embedding). You're offline."
- [ ] Offline during sync → note syncs to Notion (still works), embed deferred via best-effort
- [ ] Come back online, retry search → works normally
